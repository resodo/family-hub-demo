# AI 使用策略

> 私密文档

## 核心理念

**不是让 AI 做决策，而是让 AI 帮我维护足够好的 context，让我自己做更好的决策。**

## 当前使用场景

### P0 — 家庭健康管理（family-hub）
- Claude Code 作为主要交互入口（桌面端）
- OpenClaw 作为移动端入口（外出/碎片时间）
- 网站作为展示层（家人查看）
- **核心价值**：低摩擦地维护结构化的家庭健康 context

### P1 — 降低维护摩擦
- 目标：我只需要"说"，AI 负责"写入正确的地方"
- 已实现：对话式更新健康档案、菜谱、待办
- 待优化：
  - OpenClaw 的 Cron Job（定期提醒产检、补药等）
  - 家人直接和 AI 交互（小鱼通过 OpenClaw 查询/记录）

### P2 — 未来方向
- RAG：让 AI 能检索历史对话，回忆之前讨论过的决策 context
- 事件驱动提醒：产检前自动提醒准备事项
- 家人协作：小鱼和阿姨也能通过简单的方式和 AI 交互（不需要学 Claude Code）
- 多设备同步：iPad / 手机 / 电脑无缝切换

## 选择 Claude Code 的原因

1. **文件系统就是数据库**：markdown 文件 + git = 天然的版本管理
2. **CLAUDE.md 就是 prompt engineering**：一次写好规则，每次对话自动加载
3. **工具生态**：MCP 连接 Google Calendar、小红书等外部服务
4. **低成本**：不需要开发 app，不需要后端，就是一堆 markdown 文件
